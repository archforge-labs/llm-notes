# ベクトルと行列の直感 ── LLMの計算はほぼこれだけ

> シリーズ「LLMの中身を数式で追う」第1回

---

前回、LLMは「行列の掛け算の連鎖」だと書いた。  
では「行列」とは何か。難しそうに聞こえるが、実態はシンプルだ。

---

## ベクトル ── 数値の配列

ベクトルとは、**数値を1列に並べたもの**だ。それだけだ。

```python
v = [3.0, 1.5, -2.0, 0.8]
```

プログラマーにとっては「float の配列」と同じ。  
数学では縦に書くことが多いが、中身は変わらない。

**ベクトルが「意味を持つ」とはどういうことか？**

たとえば、食べ物を3つの軸で数値化するとする。

```
軸1: 甘さ（0〜10）
軸2: 塩辛さ（0〜10）
軸3: 脂肪分の多さ（0〜10）
```

すると：

```
チョコレート = [9.0, 0.5, 7.0]
キャラメル   = [8.5, 1.0, 6.5]
ポテチ       = [1.0, 8.0, 5.0]
```

チョコとキャラメルは「近い」、チョコとポテチは「遠い」。  
この「近さ・遠さ」が、ベクトルで意味を表現する基本的な考え方だ。

LLM では「単語の意味」をこの方法で数百〜数千次元のベクトルで表現する。  
「王」と「女王」のベクトルは近く、「王」と「りんご」は遠い。

---

## 行列 ── ベクトルの変換機

行列とは、**数値を縦横に並べた表**だ。

```
A = [[2, 0],
     [1, 3],
     [0, 1]]
```

これは「3行2列の行列」。

行列の本質的な役割は**「ベクトルを別のベクトルに変換する」**こと。

### 行列とベクトルの掛け算（行列積）

入力ベクトル `x = [1, 2]`（2次元）に行列 `A`（3行2列）を掛けると：

```
A × x = [[2×1 + 0×2],    [[2],
          [1×1 + 3×2],  =  [7],
          [0×1 + 1×2]]     [2]]
```

2次元のベクトルが3次元のベクトルに変わった。

これが行列積の正体だ。**「次元を変換する操作」**。

---

## なぜ LLM はこれを使うのか

LLM の中では、「単語の意味ベクトル」が何十もの行列を順番に通過していく。

```
入力ベクトル
  → 行列A で変換
  → 行列B で変換
  → 行列C で変換
  → ...
  → 出力（次の単語の確率）
```

それぞれの行列は「このベクトルのどの部分に注目するか」を学習した重みを持っている。  
「学習」とは、この行列の中の数値を少しずつ調整していくことだ。

---

## 内積 ── 「似ているか」を1つの数値で測る

もう一つ重要な操作がある。**内積（ドット積）**だ。

2つのベクトルの内積は、対応する要素を掛けて足し合わせるだけ。

```
a = [1, 2, 3]
b = [4, 5, 6]

a · b = 1×4 + 2×5 + 3×6 = 4 + 10 + 18 = 32
```

この値が大きいほど、2つのベクトルは「同じ方向を向いている」＝「似ている」。

### 具体例

```
「王」のベクトル   = [0.9, 0.1, 0.8]  （権威高, 女性性低, 地位高）
「女王」のベクトル = [0.8, 0.9, 0.8]
「りんご」のベクトル= [0.1, 0.0, 0.0]
```

```
「王」と「女王」の内積 = 0.9×0.8 + 0.1×0.9 + 0.8×0.8 = 1.48  ← 大きい（似ている）
「王」と「りんご」の内積= 0.9×0.1 + 0.1×0.0 + 0.8×0.0 = 0.09  ← 小さい（似ていない）
```

LLM の Attention という仕組みは、この内積を使って  
「どの単語に注目するか」を決めている。（詳しくは後の回で）

## まとめ

| 概念 | 正体 | LLMでの役割 |
|---|---|---|
| ベクトル | float の配列 | 単語・文章の意味を数値で表現 |
| 行列 | float の2次元配列 | ベクトルを別のベクトルに変換 |
| 行列積 | ベクトルの変換操作 | LLMの全計算の基本単位 |
| 内積 | 2ベクトルの類似度を1数値に | 「どの単語に注目するか」を決める |

`y = f(Wx + b)` という式がニューラルネットの全ての出発点だ。
