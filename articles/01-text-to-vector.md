# テキストがベクトルになるまで ── トークン化から埋め込みの数理

> シリーズ「LLMの中身を数式で追う」第1回

LLM に文字列を渡すと、モデルは「なんとなく意味を理解して」返事をくれる。  
では、その「理解」は計算機の中でどう表現されているのか。  
今回は入力テキストが行列演算の世界に入るまでの前処理を追う。

---

## 1. まず整数にする ── トークン化

モデルは文字列をそのまま扱えない。  
最初にやることは「テキスト → 整数の列」への変換、これがトークン化。

```
"Claude は賢い" → [36827, 374, 11often, ...]  （GPT 系 BPE の例）
```

### BPE（Byte Pair Encoding）とは

頻出する文字ペアを繰り返しマージして「単語より小さく、文字より大きい」単位を作る。

```
初期: ["C", "l", "a", "u", "d", "e"]
↓ 頻出ペア "la" をマージ
["C", "la", "u", "d", "e"]
↓ 繰り返し...
["Claude"]  ← 1 トークン
```

**なぜ文字単位でも単語単位でもないのか？**  
- 文字単位: 列が長くなりすぎる、形態論情報が失われる  
- 単語単位: 語彙外単語（OOV）が処理できない、言語ごとに辞書が必要  
- BPE: 語彙サイズを固定しながら OOV をゼロにできる。数理的には圧縮理論の応用。

---

## 2. 整数をベクトルにする ── 埋め込み（Embedding）

トークン ID が得られた。次は「各 ID に対応する実数ベクトルを引く」。

### 埋め込み行列 E

```
E ∈ R^{V × d}
```

- V: 語彙数（GPT-4 は約 100,000）
- d: 埋め込み次元（GPT-3 は 12,288、小さいモデルで 768 程度）

トークン ID i に対応するベクトルは E の i 行目を取り出すだけ。

```python
# 概念コード
token_ids = [36827, 374, ...]        # shape: (seq_len,)
x = E[token_ids]                      # shape: (seq_len, d)
```

これは「ルックアップテーブルを引く」操作だが、行列積で書き直せる。

```
x = one_hot(token_ids) @ E
```

one_hot(i) は i 番目だけ 1 の V 次元ベクトル。@ E で i 行目を取り出す。  
**なぜ行列積で書くか？** ── GPU は行列積が最も速い。ルックアップより行列積に統一するとハードウェア効率が上がる。

### E の中身は何か

学習前は乱数。学習を通じて、意味的に近い単語が近くなるように調整される。

```
cos_sim(E["王"], E["女王"])  >  cos_sim(E["王"], E["りんご"])
```

有名な king − man + woman ≈ queen という加法的関係は、  
この空間が「意味の差分」を線形に保持していることを示す。

---

## 3. 位置情報を足す ── Positional Encoding

埋め込みだけでは「単語の順序」が消える。  
「犬が猫を追う」と「猫が犬を追う」は同じ埋め込みの集合になってしまう。

### サイン・コサイン PE（オリジナル Transformer）

```
PE(pos, 2i)   = sin(pos / 10000^{2i/d})
PE(pos, 2i+1) = cos(pos / 10000^{2i/d})
```

**なぜ三角関数か？**  
1. pos に対して周期的 → 相対位置の差分が内積で取り出せる  
2. `sin(a+b) = sin(a)cos(b) + cos(a)sin(b)` という加法定理が成立するため、  
   「pos から k 離れた位置」を線形変換で表現できる  

つまり Attention が Q・K の内積で位置関係を計算できるよう、  
位置情報を「線形変換で取り扱える形」に埋め込んでいる。

---

## まとめ ── 入力がモデルに入るまで

```
テキスト
  ↓ BPE トークン化
整数列 [i₁, i₂, ..., iₙ]
  ↓ 埋め込み行列 E でルックアップ
実数行列 X ∈ R^{n × d}
  ↓ Positional Encoding を加算
X' ∈ R^{n × d}  ← ここから Transformer 本体へ
```

次回は Transformer 本体の最初の操作、**Self-Attention** を追う。  
Q・K・V という3つの行列が「どのトークンに注目するか」を数式でどう決めているのか。
