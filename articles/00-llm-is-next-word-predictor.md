# LLMは「次の単語を予測する機械」にすぎない

> シリーズ「LLMの中身を数式で追う」第0回

---

ChatGPT や Claude に話しかけると、まるで「考えて」答えてくれるように見える。  
でも、その裏側にある動作原理は驚くほどシンプルだ。

**LLM がやっていることは、ひとつだけ。**

> 「ここまでの文章の続きとして、次に来る単語は何か」を予測する。

これだけだ。

---

## 具体例で見てみる

たとえば、次の文章があるとする。

```
東京の天気は今日、
```

LLM はこの入力を受け取って、次に来うる単語の「確率リスト」を計算する。

```
「晴れ」  → 42%
「雨」    → 31%
「曇り」  → 18%
「雪」    → 5%
「...」   → ...
```

そしてこのリストから1つを選んで出力する。  
選び方は「最も確率が高いものを選ぶ」か、「確率に従ってランダムに選ぶ」かのどちらか。

「晴れ」が選ばれたとする。次は：

```
東京の天気は今日、晴れ
```

この文章に対してまた同じことをする。

```
「で」   → 55%
「です」 → 30%
「でした」→ 10%
...
```

これを繰り返す。それだけだ。

---

## なぜこれだけで「賢く見える」のか

一見、「次の単語を当てるだけ」なら大したことはないように思える。  
しかし考えてみてほしい。

「東京の天気は今日、晴れで」の次に自然に来る単語を正確に予測するためには、  
LLM は以下のことを「知っている」必要がある。

- 「天気」という概念の意味
- 「晴れ」の後に続く日本語の文法パターン
- 天気についての一般常識
- この文脈でどんな表現が自然か

つまり、**「次の単語を正確に予測できる」ということは、「文章の意味を理解している」ことと限りなく近い。**

膨大なテキストデータで「次の単語予測」を繰り返し学習した結果、  
モデルは言語の構造・文脈・知識を自然に獲得していく。

---

## 「確率リスト」はどうやって計算するのか

ここからが、このシリーズで追っていく本題だ。

入力テキストを受け取って「次の単語の確率リスト」を出力するまでの計算は、  
すべて**行列の掛け算**でできている。

```
テキスト → [行列の掛け算の連鎖] → 確率リスト
```

その行列の掛け算をどう設計すれば「賢い予測」ができるのか。  
それが Transformer というアーキテクチャの正体だ。

次回は、その行列の掛け算の話をする前に、  
「ベクトル」と「行列」が何者なのかを直感で掴む。

---

## まとめ

- LLM は「次の単語の確率」を計算する機械
- その計算を繰り返すことで文章が生成される
- 「次の単語を正確に予測できる」≈「言語を理解している」
- 内部の計算はすべて行列演算
